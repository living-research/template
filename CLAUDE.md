# CLAUDE.md

## Project

Living Research project. Notebooks are the primary research artifact. The published page reads data that notebooks produce.

## Structure

```
notebooks/         # Jupyter notebooks — run end-to-end, produce all outputs
scripts/           # Reusable utilities (colab_utils.py for Colab publish flow)
data/raw/          # Source data (immutable)
data/processed/    # Derived data (regenerated by notebooks, gitignored)
results/           # Outputs: plots, models, intermediate files (gitignored)
docs/              # Published findings → GitHub Pages
docs/data/         # Data the findings page reads at runtime
```

## Data flow

`data/raw/` (immutable) → notebook → `data/processed/` + `results/` + `docs/data/` → publish cell pushes to GitHub → Pages deploys `docs/`

## Commands

```bash
make run       # Execute all notebooks locally via nbconvert
make clean     # Delete data/processed/* and results/*
```

## Workflow

1. Open notebook in Colab (or run locally)
2. Notebook runs full pipeline: load data, analyze, write outputs to `results/` and `docs/data/`
3. Publish cell in notebook commits and pushes artifacts back to GitHub
4. GitHub Pages deploys updated findings page automatically

## CI

- **validate.yml** — runs notebooks on push to verify they still execute. Validation, not primary execution.
- **publish.yml** — deploys `docs/` to GitHub Pages when `docs/` changes.

## Conventions

- Raw data is immutable.
- One notebook per analysis. Dependencies in `requirements.txt`.
- Commit messages document reasoning — git history is the research record.
- Dual license: CC BY 4.0 for text/findings, MIT for code.

## Design System (docs HTML)

CSS custom properties defined in `docs/index.html`:
- Colors: `--text`, `--text-secondary`, `--bg` (#fafaf8), `--surface` (#f5f5f0), `--accent` (#115e59 teal)
- Spacing: `--space-xs` through `--space-2xl` (asymmetric: 3.5rem, 5rem)
- Max width: `--measure` (38rem)
- Typography: Georgia serif for body, system font stack for UI/labels
- Auto dark mode via `prefers-color-scheme: dark`
